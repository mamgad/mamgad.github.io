How I turned my scattered brain into a productivity powerhouse using AI and some clever automation

The Scattered Brain Problem: My Brain vs. Traditional Tools
#

Here’s the deal with my scattered brain: it works in patterns that traditional project management tools completely ignore. I’m juggling multiple complex projects, and every PM tool I tried felt like trying to fit a square peg in a round hole.

The core issues:

  • 🧠 Context switching kills me: Jumping between tools breaks my flow state
  • 📝 I forget everything: What was I working on? Why did this matter?
  • 🔥 Hyperfocus then chaos: I’ll code for 6 hours straight, then lose track for days
  • 📊 Manual updates? Yeah right: If it’s not automatic, it’s not happening

What I needed was basically an AI assistant that could:

  1. Remember what I was doing and why
  2. Track my progress without me thinking about it
  3. Help me prioritize when my brain is all over the place
  4. Work with my chaotic workflow, not against it

The Solution: An AI Assistant That Actually Gets It
#

I built my own AI assistant using Claude Code + Obsidian. Think of it as having a really smart teammate who never sleeps, never forgets context, and actually understands how chaotic brains work.

Step 1: Obsidian as My External Brain
#

MyVault/
├── Daily Notes/     # Brain dumps and daily check-ins
├── Projects/        # Multiple work projects
├── Templates/       # Consistent structure
└── scripts/         # Automation magic

The key insight: Structure without rigidity. I can dump thoughts anywhere, and the system helps me organize later.

The magic part? When I ask Claude to create tasks and projects, Obsidian’s graph view automatically shows all the connections between my notes, tasks, and projects. No manual linking required - just ask Claude to organize something, and boom, the visual web of my work appears.

Note: Actual project and task names are hidden for privacy. All examples in this post use dummy data.

Step 2: Claude Code as My AI Sidekick
#

This is where the magic happens. I created custom slash commands so my AI assistant can:

/show-tasks - “What should I work on?”

  • Sorts everything by urgency and priority
  • Shows me what’s actually on fire vs what can wait
  • No more staring at endless lists wondering where to start

/daily-check-in - My team communication automation:

  • Automatically detects what I did (using git commits!)
  • Generates daily standup updates for my team
  • Formats accomplished tasks, current work, and blockers
  • Creates both detailed and Slack-ready formats for easy sharing

Step 3: The Task Format That Saved My Sanity
#

- [ ] Fix login bug @task-20250810-001 (due:: 2025-08-16) (priority:: P1) (project:: WebApp)
  - status:: not-started
  - estimate:: 2d
  - notes:: Users can't authenticate, breaks entire flow

Why this works for scattered brains:

  • Unique IDs: Nothing gets lost in my brain fog
  • Machine readable: My AI can sort and prioritize automatically
  • Context preserved: I remember WHY things matter

Step 4: Git as My Accountability Partner
#

The genius move: git commits don’t lie about what I actually did.

git log --since="yesterday" --oneline  # Shows real progress
git diff HEAD~1 -- Projects/          # Detects completed tasks

No more guilt about “what did I even do today?” - git remembers everything.

How It Actually Works: My Daily Flow
#

Morning: “/daily-check-in”
#

My AI generates my daily standup update by:

  • Analyzing git commits to show what I accomplished yesterday
  • Listing what I’m working on today (sorted by priority)
  • Capturing any blockers or challenges I’m facing

The output is formatted for easy sharing with my team - no more scrambling to remember what I did when standup starts.

During Work: “/show-tasks”
#

When my brain goes “wait, what was I doing?” - I just ask my AI. It shows me everything sorted by urgency, no thinking required.

Evening: Automatic Progress Tracking
#

Every commit I make gets tracked. No manual updates, no guilt about forgetting to log progress.

The ADHD-Friendly Workflow
#

Brain Dump → Organization → Action
#

  1. Messy thoughts go in daily notes (zero friction)
  2. AI assistant helps organize when I’m ready
  3. Smart prioritization tells me what actually matters
  4. Automatic tracking keeps me accountable

What I Learned: ADHD-Friendly Design Principles
#

1. Reduce Cognitive Load
#

  • One place to dump all thoughts
  • AI figures out organization later
  • Clear visual priorities (red = urgent, green = chill)

2. Automate the Boring Stuff
#

  • Git commits = automatic progress tracking
  • No manual status updates required
  • Smart reminders based on actual behavior

3. Work WITH ADHD, Not Against It
#

  • Hyperfocus: Let me code uninterrupted, track everything automatically
  • Scattered: Clear, simple prioritization when I need direction
  • Forgetful: Context preservation so I never lose track

The Results: My ADHD Brain Finally Has A Co-Pilot
#

Before: Scattered thoughts, forgotten tasks, constant guilt about productivity

After: Clear priorities, automated tracking, an AI assistant that actually gets my workflow

The Real Impact
#

  • No more lost context: I can hyperfocus for hours, then instantly remember where I was
  • Guilt-free breaks: Git history shows I actually am productive, even when it doesn’t feel like it
  • Clear direction: When my brain is scattered, /show-tasks tells me exactly what matters
  • Automatic accountability: Progress tracking happens in the background

Why This Works for ADHD
#

Traditional productivity advice: “Just use a planner and stick to it!”

ADHD reality: My brain doesn’t work that way.

This system works because it:

  1. Meets me where I am (scattered, forgetful, hyperfocused)
  2. Automates the boring stuff (tracking, organizing, prioritizing)
  3. Preserves context (so I never lose the “why”)
  4. Provides clear direction (when I need it)
  5. Gets out of my way (when I don’t)

Live Examples: See the System in Action

Smart Task Creation Demo
#

Here’s what happens when I casually mention a new task:

Me: "My manager gave me a new task to build a PoC for a quantum calculator"

Claude: I'll help you create a new task for the quantum calculator PoC. 
First, let me check what projects you have and create the appropriate 
project structure.

I need to ask you a few questions before creating the task:

1. **Due date:** When does your manager need this quantum calculator 
   PoC completed?
2. **Priority:** How urgent is this compared to your other work? 
   (P1=Critical, P2=High, P3=Normal, P4=Low)
3. **Project:** Should this be part of an existing project (CyWAF, Nexus) 
   or should I create a new "QuantumCalculator" project for it?

Once you provide these details, I'll create the properly formatted task 
using the hybrid format and generate the next available task ID.

Why this is genius: Claude automatically asks for missing critical information (due date, priority, project) instead of creating incomplete tasks. No more “when was this due again?” moments later.
